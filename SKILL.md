# 中文科技报告写作规范

## 一、核心写作风格规范

### 1.1 语体要求

全文采用正式严谨的书面科技语体，句式平实客观，逻辑连贯条理清晰，贴合工程技术、科研报告通用撰写规范。

- 句式为陈述句主导，使用客观中性的学术表达
- 逻辑关系通过连词和语序自然呈现，不靠标点符号拼接
- 每段应有明确的主题句，段与段之间通过逻辑过渡衔接
- 段落层次划分合理，前后行文语气统一

### 1.2 禁止事项

**禁止AI模板化、套路化表达**，包括但不限于：

- "随着XX的发展""在XX的背景下，XX应运而生""XX扮演着重要的角色""不可忽视的是""值得一提的是""XX备受关注""引起了广泛关注"等空话套话
- "总的来说""综上所述"等冗余过渡词
- "具有重要的意义""发挥着关键作用"等虚化评价

**禁止不规范标点使用**：

- 禁止随意添加双引号进行强调或隐喻
- 禁止使用多余破折号、波浪线等标点符号
- 最大限度规避所有括号注释形式的补充说明
- 能用陈述句直接表述的内容绝不使用括号补充
- 括号仅用于文献引用编号标注

**禁止冗余表达**：

- 不使用口语化、网络化、碎片化表述
- 不套用固定句式、不使用浮夸修饰词
- 不做冗余空话铺垫，开门见山
- 句子长度适中，不刻意使用长难句堆砌复杂度

### 1.3 专业术语规范

- 全文专业术语表述规范统一
- 同一术语在全文保持一致的中文译名
- 首次出现的专业术语可括注英文全称和缩写
- 后续出现统一使用中文术语或首次确定的缩写形式
- 不篡改原有技术参数、研究结论、流程步骤及专业术语

## 二、反AI检测控制规范

### 2.1 AI率控制目标

- 严格控制文本AI重复率及机器文风
- 最终成品AI检测率必须控制在5%以内
- 语言贴近人工手写科技报告的自然表述习惯

### 2.2 具体控制措施

**句式多样性**：
- 同一段落内相邻两个句子不使用相同的句式结构
- 避免"XX是XX的""XX是指XX""XX表示XX"等判断句连续堆砌
- 交替使用主动句和被动句
- 合理使用长短句交替，避免全篇等长句式

**用词多样性**：
- 同一段落内避免重复使用相同的过渡词和逻辑连词
- 表示因果关系的词汇交替使用：因此、从而、据此、由此、基于此
- 表示转折的词汇：然而、但、不过、与之不同、相对而言
- 避免连续使用"的"字结构

**段落结构多样性**：
- 不套用固定的段落展开模板
- 每段的展开方式各有不同
- 综合运用定义解释、对比分析、因果推演、举例说明、数据支撑等展开方式

### 2.3 自然化处理

- 行文保持人工写作的自然节奏感
- 适当使用承前省略等自然语言特征
- 逻辑连接点到为止，不过度使用连词
- 段首句不套用固定模式

## 三、修改润色专项规范

### 3.1 保留内容

- 保留原有文章结构，不改变章节标题体系和整体框架
- 保留核心专业原意，不改变技术观点和立场
- 保留专业技术参数、实验数据、计算公式
- 保留研究结论和核心判断
- 只做优化完善不做颠覆性改写

### 3.2 需要修改的内容

**语句优化**：
- 修正语病和语序混乱的问题
- 删除重复赘述的内容
- 简化冗余句式，一句话能说清的不用两句
- 修正主语不明、指代不清的表述

**逻辑补充**：
- 补充必要的逻辑过渡语句
- 补齐内容残缺部分（明显缺失的推理步骤或过渡段落）
- 完善段与段之间的逻辑衔接
- 强化前提与结论之间的推理链条

**专业化调整**：
- 将口语化表达转为书面科技语体
- 统一全文专业术语表述
- 修正不规范的计量单位和符号
- 补充必要的技术限定条件

### 3.3 不改动的情形

- 作者原文中清晰的表述不做无意义改写
- 已合乎规范的专业术语不替换
- 作者特有的、符合规范的行文风格予以保留
- 不需要为了凑字数而扩写，也不需要为精简而删除实质内容

## 四、写作流程

### 第一阶段：文档分析
- 阅读并理解用户提供的Word文档或文字内容，掌握文章主旨和技术脉络
- 识别文档类型（综述、研究报告、实验报告、课程论文等）
- 评估需要修改的程度（仅润色、轻度改写、深度优化、扩写补充）

### 第二阶段：问题诊断
- 逐段扫描，标记具体问题点：
  - 语病和语序混乱
  - 逻辑断裂或跳跃
  - 重复赘述
  - 口语化表达
  - 内容残缺
  - AI套路化表达
  - 括号滥用

### 第三阶段：逐段修改
- 按原有章节结构逐段处理
- 每段修改优先顺序：修正语病→完善逻辑→规范表达→精简冗余→统一术语
- 修改后通读全段，检查表达自然度和AI率
- 确保不改变原文技术含义

### 第四阶段：全文通读
- 检查全文语气统一性
- 检查各章节篇幅比例是否合理
- 检查专业术语前后一致性
- 检查逻辑链是否完整贯通

## 五、各章节写作细则

### 5.1 摘要
- 摘要应涵盖：研究背景、现存问题、本文工作、主要结果与结论
- 每部分1-2句，层次清晰
- 不使用"本文""本研究"以外的一人称
- 不使用评价性语言
- 关键词3-6个

### 5.2 引言
- 阐述研究背景和技术现状，1-2段
- 明确指出存在的问题或技术瓶颈
- 说明本文目的和结构安排
- 引言应简洁聚焦，不展开过多技术细节

### 5.3 技术原理与方法
- 按逻辑层次从基础到深入展开
- 每个技术点用独立段落展开
- 结合具体数据和技术参数
- 适当引用文献支撑

### 5.4 现状与分析
- 按技术分类或时间顺序组织
- 每种技术独立成段或成节
- 包含技术原理、发展现状、优劣对比三个层次
- 对比分析应客观，避免主观偏好

### 5.5 挑战与展望
- 挑战应具体、有针对性，不写泛泛之谈
- 展望应基于现有技术趋势合理推演
- 每个要点用1-2段展开

### 5.6 结论
- 概括全文核心工作
- 总结主要发现和结论
- 展望未来研究方向
- 不与摘要简单重复

### 5.7 参考文献
- 采用顺序编码制，按引用顺序编号
- 格式遵循GB/T 7714标准
- 引用标注置于句末标点之前
- 参考文献列表按规范格式排列

## 六、模板体系与填充规程

### 6.1 模板类型索引

本技能内置以下7类科技报告模板，存放于 templates/ 目录下：

| 模板编号 | 模板名称 | 适用场景 |
|---------|---------|---------|
| T01 | 课程报告 | 研究生/本科生课程结课报告 |
| T02 | 文献综述 | 研究领域技术综述 |
| T03 | 实验报告 | 科学实验或工程实验 |
| T04 | 技术研究报告 | 项目技术研究与分析 |
| T05 | 学位论文大纲 | 本科/硕士/博士学位论文框架 |
| T06 | 项目结题报告 | 科研项目结题总结 |
| T07 | 技术调研报告 | 技术选型与方案对比 |

### 6.2 模板变量体系

所有模板统一使用变量占位符体系，填充时根据用户提供的信息撰写为完整段落。

**全局元变量**（所有模板通用）：{{课程名称}}、{{学生姓名}}、{{学号}}、{{专业}}、{{学院}}、{{学校}}、{{指导教师}}、{{提交日期}}、{{总字数}}

**内容变量**（按模板类型选用）：{{研究背景}}、{{研究目的}}、{{国内外现状}}、{{核心原理}}、{{方法步骤}}、{{实验结果}}、{{技术对比}}、{{问题分析}}、{{解决方案}}、{{核心结论}}、{{未来展望}}、{{参考文献列表}}

### 6.3 模板填充流程

当用户要求"按模板写"或指定报告类型时，执行以下流程：

**第一步：匹配模板。** 根据用户描述的报告类型选择对应的模板。如用户未明确指定类型，引导用户确定模板类型。

**第二步：收集变量。** 获取全局元变量。对于内容变量，基于用户的技术方向或研究主题，结合专业知识填充，填充内容须遵循本规范全部写作要求。

**第三步：逐项填充。** 按照模板的章节结构逐节撰写，每个变量占位符展开为完整段落。段落本身须符合核心写作风格规范及反AI检测控制规范。

**第四步：全文整理。** 统一章节编号格式，统一图表编号格式（图X.Y、表X.Y），统一参考文献引用格式。如用户要求则生成目录。

### 6.4 模板定制规则

- 用户可要求对模板进行结构调整（增删章节、调整顺序）
- 用户可要求增加特定内容段落
- 用户可指定字数分配比例
- 模板调整后仍需保证全文逻辑连贯、结构完整

### 6.5 章节字数分配参考

对于一份完整的科技报告，各章节建议字数占比：

| 章节 | 占比 | 8000字示例 | 15000字示例 |
|------|------|-----------|------------|
| 摘要与关键词 | 3%-5% | 300字 | 500字 |
| 引言 | 10%-15% | 1000字 | 1800字 |
| 原理与方法 | 20%-30% | 2000字 | 4000字 |
| 现状与分析 | 25%-35% | 2500字 | 4500字 |
| 挑战与展望 | 10%-15% | 1000字 | 2000字 |
| 结论 | 3%-5% | 300字 | 500字 |
| 参考文献 | 不计入 | 15-30篇 | 25-50篇 |

## 七、响应策略

### 7.1 用户上传Word文档要求修改
→ 按第三部分修改润色规范执行，读取文档内容后逐段处理

### 7.2 用户要求撰写新报告
→ 询问报告类型和字数要求，匹配对应模板，按模板填充流程执行

### 7.3 用户要求按模板写作
→ 确认模板类型，按6.3节模板填充流程逐项执行

### 7.4 用户要求润色已提供文字
→ 直接对提供文字进行问题诊断和逐句优化

### 7.5 用户要求扩写补充
→ 在现有框架下补充内容，新增内容严格遵循本规范

## 八、质量控制清单

每完成一次写作任务后自查：

- [ ] 全文是否存在AI套路化表达
- [ ] 是否存在不必要的括号注释
- [ ] 是否存在不当的双引号或破折号
- [ ] 专业术语是否前后统一
- [ ] 句式是否足够多样
- [ ] 段落衔接是否自然
- [ ] 技术参数和结论是否被篡改
- [ ] 全文语气和风格是否一致
- [ ] AI检测率是否在5%以下
- [ ] 是否保留原文核心结构和原意
- [ ] 模板填充是否完整无遗漏
- [ ] 章节字数分配是否合理
